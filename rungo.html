DESCRIPTION

The project is to design a new feature for new and intermediate users. Rungo is an existing mobile application that allows the user to find, create and follow a new running route by turn-by-turn navigation. Our team decided to design a new feature for Rungo to give it a higher engaging rate between the user and Rungo. To achieve our goal, we approach with lateral thinking to solve problems using indirect and creative ways with reasons stated clearly.

1. UNDERSTANDING RUNGO

Before actually designing the new feature for Rungo, it is important to complete some exploratory and evaluative research to understand Rungo's existing interface, interaction patterns, features, and users, with its core language, patterns, and workflows.

GOAL

RunGo's purpose is to make runners lives easier and safer by providing voice navigation, and a route for the runners to follow.

INTENDED USER

The runners who need a safe way to jog on the street with path navigated and have a record of statistics on health, pace, etc.

FEATURES

Run Analytics

Track your essential running stats like time, pace, splits, distance and elevation.

Turn-by-turn Navigation

Turn-by-turn navigation allows the user to enjoy his run, and take away the concern of finding his way back home.

Global Running Routes

The runner can find a run suited to his needs any- where in the world. It also includes 200 new routes added daily and verified and curated routes built by RunGo and certified local run groups.

2. NEW FEATURE - ROUTE DETAILS UPDATE

By updating the route details, when users choose the routes or create their route, the application will provide the current weather, air fresh level, peak information of the current time.

INFORMATION CATEGORIZED

Information is divided into two categories. Under peak hour, it shows the data of both human flow and traffic flow. Under weather, it shows the information of a specific route/area’ temperature, weather, humidity, air quality, risk level and more.

WEATHER UPDATE

A pop-up notification that shows weather update is provided to give the user a head up for weather changing and safer running experience.

WEATHER UPDATE

The homepage has a section on the top that includes the basic information of the user’s current location weather. This gives a better user experience to the user which avoids them to create/decided on a route and found out the weather is not perfect for running.

NEW FEATURE FLOW

The interaction is mainly by tapping and swiping.

CHALLENGES + REFLECTION

While deciding on what to design as a new feature, our team come up with three ideas. As one of the ideas was to target an office lady with a tight schedule. We were thinking of having the Rungo application to be an application that helps connect the users with his family and friends. The goal was to help the user with a tight schedule could still have a high participating rate by combining his daily life ( shopping time with friends, drinking time with friends ) to the running routine. So, the user can run with friends and save time at the same time. However, making it a social application could be hard to maintain, and it is not what Rungo wants as a running application. We overcome the problem with iterating more deeply in the other ideas. So, we can focus on target audiences with a better solution and create a better scope.
